**Recommendation:**

### 2. Model Prediction Shows `nan` Values
**Evidence:**
```
2026-03-10 07:30:50 | ModelManager | INFO | Prediction (scaled=-0.033977, inverse=nan, input_dim=25)
```

**Root Cause:** Inverse transform (scaler) is producing NaN, likely due to:
- Scaler not fitted before predict
- Input data contains NaN/Inf values
- Mismatch between training and inference feature columns

**Recommendation:**
- Add input validation before prediction
- Log scaler statistics (mean/std) for debugging
- Add `np.isfinite()` checks on model outputs
