# Kiro Quant V3.6 Codex 實施任務清單
## 根據《量化交易》書籍重新編排的策略

**策略文件**: `V3.6_STRATEGY_DESIGN.md`

---

## 🎯 優先任務順序

### 🔴 Phase 1: 核心資金管理 (最高優先)

#### 任務 1: Kelly Formula 實現
```
檔案: v3_pipeline/risk/kelly_sizer.py (新建)
功能:
- 計算最佳倉位大小
- 支援半 Kelly 模式
- 風險限制 (最大 20% 倉位)

測試:
- win_rate=0.55, avg_win=100, avg_loss=80 → position ≈ 6.875%
```

#### 任務 2: 風險規則引擎
```
檔案: v3_pipeline/risk/rules.py (新建)
規則:
- 每筆交易 ≤ 2% 風險
- 單日 ≤ 5% 風險
- 最大 10 隻持倉
- 最大 80% 總倉位
```

---

### 🟡 Phase 2: 交易成本系統

#### 任務 3: Transaction Cost Calculator
```
檔案: v3_pipeline/execution/cost_calculator.py (新建)
功能:
- 佣金計算 (US: 0.1%, HK: 0.1% + 印花稅)
- 滑點模型 (大型股 0.05%, 中型 0.1%, 小型 0.2%)
- 總成本計算
```

#### 任務 4: 成本門檻檢查
```
功能:
- 策略收益 < 成本 → 不執行
- 回測必須扣除成本
```

---

### 🟡 Phase 3: 風險度量系統

#### 任務 5: VaR/CVaR 計算器
```
檔案: v3_pipeline/risk/metrics.py (新建)
功能:
- VaR 95% 計算
- CVaR 95% 計算
- 最大回撤追蹤
```

#### 任務 6: 風險控制規則
```
觸發條件:
- VaR 報警: 暫停交易
- 回撤 > 15%: 停止新交易
- 回撤 > 20%: 只平倉
- 回撤 > 30%: 策略暫停
```

---

### 🟢 Phase 4: 評估指標

#### 任務 7: Performance Metrics
```
檔案: v3_pipeline/analysis/metrics.py (新建)
指標:

- Information Ratio

- Sortino Ratio
- Win Rate
- Profit Factor
- 最大回撤
```

---

### 🔵 Phase 5: 策略擴展

#### 任務 8: 市場狀態識別
```
檔案: v3_pipeline/strategies/regime_detector.py (新建)
狀態:
- trending_up (趨勢上升)
- trending_down (趨勢下降)

根據狀態自動切換策略
```

#### 任務 9: 多策略支援
```
支援:
- 均值回歸 (RSI)
- 動量 (MACD)
- 突破策略
- 配對交易
```

### 🟣 Phase 6: 回測框架

#### 任務 10: Backtest Validator

檔案: v3_pipeline/backtest/validator.py (新建)
防止偏差:
