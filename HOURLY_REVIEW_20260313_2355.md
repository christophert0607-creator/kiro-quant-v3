### Technical Details - Backoff Reset Bug

**Location:** `v3_pipeline/core/futu_connector.py` lines 287-303

```python
def reconnect(self, max_retries: int = 10, base_delay_seconds: int = 5) -> None:
    last_exc: Optional[Exception] = None
    for attempt in range(1, max_retries + 1):  # <-- Resets each call!
        try:
            self.logger.warning("Reconnect attempt %d/%d", attempt, max_retries)
            self._safe_close_contexts()
            self.connect()
            ...
            delay = base_delay_seconds * (2 ** (attempt - 1))  # <-- Starts fresh each call
```

**Problem:** Each call to `reconnect()` starts with attempt=1, meaning:
- First call: 5s, 10s, 20s, 40s... (max 10 attempts)
- Second call (after failure): **resets to 5s, 10s, 20s...**
- This creates a tight loop that never reaches longer delays

**Called from:** `main_loop.py:540-547` in `_attempt_reconnect()`
- Called on every heartbeat failure
- No upper limit on total reconnect attempts
